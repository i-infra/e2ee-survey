How It Works

How End-to-End Encryption Works

  • Survey Creation: You write your survey in markdown and set a password
  • Client-Side Encryption: Your survey is encrypted in your browser before uploading
  • Secure Storage: Only encrypted data is stored on our servers
  • Response Collection: Respondents decrypt the survey, answer, and re-encrypt their responses
  • Analysis: Only you can decrypt and view the responses with your password

Privacy Guarantees

  • Zero-knowledge architecture - we cannot decrypt your data
  • No user accounts or personal information required
  • Surveys are automatically cleaned up after 30 days via lazy cleanup system
  • All encryption happens in your browser using proven algorithms
  • Open source and auditable code
  • Cleanup runs when analysis pages are accessed (1-hour cooldown prevents abuse)

Privacy First: This tool was built as an alternative to traditional survey platforms that require accounts and may access your data. Here, privacy is not optional - it's built into the architecture.


Technical Details:

  • Encryption: TweetNaCl's XSalsa20-Poly1305 authenticated encryption (32-byte keys, 24-byte nonces)
  • Key Derivation: Argon2id with 32-byte output, 16-byte random salt, 3 iterations, 64MB memory
  • Key Verification: SHA-256 hash of derived key for password verification
  • Random Generation: Cryptographically secure random bytes for salts and nonces
  • Data Format: Encrypted data stored as Uint8Array, converted to/from Base64 for transport
  • Infrastructure: Cloudflare Workers edge computing with D1 SQLite database
  • Transport Security: HTTPS-only with CORS headers for cross-origin protection
